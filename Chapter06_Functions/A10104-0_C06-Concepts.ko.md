## 1. 함수 개념

### 1.1 함수의 필요성

**함수**는 특정 작업을 수행하는 이름이 붙은 재사용 가능한 코드 블록이다. 함수가 없으면 프로그램이 길고, 반복적이며, 유지보수가 어려워진다. 함수는 네 가지 핵심 이점을 제공한다:

| 이점 | 설명 |
|:----|:----|
| **코드 재사용** | 한 번 작성하고 여러 번 호출 — 복사/붙여넣기 불필요 |
| **모듈화** | 큰 문제를 작고 관리 가능한 조각으로 분리 |
| **가독성** | `main()`이 목차처럼 읽히고, 세부 사항은 함수 내부에 숨겨짐 |
| **유지보수성** | 하나의 함수에서 버그를 수정하면 모든 호출 지점이 자동으로 수정됨 |

**변경 전 — 모든 것이 `main()` 안에:**

```c
#include <stdio.h>

int main() {
    // --- 5의 팩토리얼 계산 ---
    int fact1 = 1;
    for (int i = 1; i <= 5; i++) {
        fact1 *= i;
    }
    printf("5! = %d\n", fact1);

    // --- 8의 팩토리얼 계산 (중복된 로직!) ---
    int fact2 = 1;
    for (int i = 1; i <= 8; i++) {
        fact2 *= i;
    }
    printf("8! = %d\n", fact2);

    return 0;
}
```

**변경 후 — 로직을 함수로 추출:**

```c
#include <stdio.h>

int factorial(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

int main() {
    printf("5! = %d\n", factorial(5));
    printf("8! = %d\n", factorial(8));
    return 0;
}
```

중복된 반복문이 사라졌다. 팩토리얼 알고리즘을 변경해야 할 때 `factorial()`만 수정하면 된다.

### 1.2 함수의 종류

C 프로그램에서는 세 가지 범주의 함수를 사용한다:

```
C의 함수
├── 1. 진입점 함수      ← main()
├── 2. 라이브러리 함수   ← printf(), scanf(), sqrt(), ...
└── 3. 사용자 정의 함수  ← 프로그래머가 직접 작성
```

| 분류 | 예시 | 비고 |
|:----|:----|:----|
| 진입점 | `main()` | 모든 C 프로그램에는 정확히 하나의 `main()`이 있어야 한다. 실행이 여기서 시작된다. |
| 라이브러리 | `printf()`, `scanf()`, `sqrt()` | C 표준 라이브러리에서 제공. 적절한 헤더(`<stdio.h>`, `<math.h>` 등)를 `#include`해야 한다. |
| 사용자 정의 | `factorial()`, `isLeapYear()` | 프로그래머가 특정 하위 문제를 해결하기 위해 작성. |

> **핵심:** `#include <stdio.h>`는 함수 코드 자체를 포함하는 것이 **아니다**. 컴파일러가 함수 호출 방법을 알 수 있도록 **선언(원형)**을 포함하는 것이다. 실제 구현은 나중에 링커가 연결한다.

### 1.3 함수 사용의 세 가지 요소

C에서 함수를 사용하려면 세 가지 요소가 필요하다:

```
함수 사용의 세 가지 요소
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   정의         │    │   선언(원형)    │    │      호출       │
│  (Definition) │    │ (Declaration) │    │    (Call)     │
│               │    │               │    │               │
│  함수가 하는 일  │    │ 함수의 존재를 약속 │    │ 실제로 함수를 실행 │
└───────────────┘    └───────────────┘    └───────────────┘
```

| 요소 | 목적 | 필수 여부 |
|:----|:----|:---------|
| **정의** | 함수를 구현 (헤더 + 본체) | 반드시 정확히 한 번 |
| **선언** (원형) | 함수의 존재를 컴파일러에 알림 (반환형, 이름, 매개변수) | 호출이 정의 **앞에** 오는 경우 필요 |
| **호출** | 특정 인수로 함수를 실행 | 사용하려면 반드시 필요 |

## 2. 함수 기본

### 2.1 함수 정의

함수 정의는 **헤더**와 **본체**로 구성된다.

```
┌─── 함수 헤더 ────────────────────────────┐
│  반환형   함수이름(매개변수 목록)             │
└────────────────────────────────────────┘
┌─── 함수 본체 ──────┐
│  {               │
│      문장들;       │
│      return 값;   │
│  }               │
└──────────────────┘
```

**주석이 달린 예제:**

```c
/*  반환형
 *    │
 *    │    함수 이름
 *    │       │
 *    │       │        매개변수 목록
 *    │       │            │            */
    int     factorial   (int n)          // ← 헤더
    {                                    // ← 본체 시작
        int result = 1;                  //   지역 변수
        for (int i = 1; i <= n; i++) {
            result *= i;
        }
        return result;                   //   호출자에게 값 반환
    }                                    // ← 본체 끝
```

| 구성 요소 | 설명 |
|:---------|:----|
| 반환형 | 호출자에게 돌려보내는 값의 데이터 타입 (`int`, `double`, `void`, ...) |
| 함수 이름 | 명명 규칙을 따르는 식별자 (2.3 참조) |
| 매개변수 목록 | 호출자로부터 값을 전달받는 `타입 이름` 쌍의 쉼표 구분 목록 |
| 본체 `{}` | 함수가 호출될 때 실행되는 코드 블록 |

### 2.2 반환형

반환형은 함수가 돌려보내는 값의 종류를 지정한다.

| 반환형 | 의미 | 예시 |
|:------|:----|:----|
| `int` | 정수를 반환 | `int factorial(int n)` |
| `double` | 실수를 반환 | `double getAverage(int a, int b)` |
| `void` | 아무것도 반환하지 않음 | `void printMenu(void)` |

```c
// int 반환형
int square(int x) {
    return x * x;
}

// double 반환형
double average(int a, int b) {
    return (double)(a + b) / 2;
}

// void 반환형 — 반환값 없음
void printLine(void) {
    printf("--------------------\n");
}
```

**중요한 규칙:**

1. **함수는 하나의 값만 반환할 수 있다.** 여러 결과를 반환하려면 포인터가 필요하다 (8장에서 다룸).
2. 반환형을 생략하면 컴파일러가 `int`로 간주한다 (C89 동작). **항상 반환형을 명시적으로 작성하라** — 생략은 더 이상 권장되지 않으며 오류의 원인이 된다.
3. `void` 함수는 `return;` (값 없이)을 사용하여 조기에 종료하거나, 닫는 `}`까지 실행이 흘러가도록 할 수 있다.

```c
void printIfPositive(int n) {
    if (n <= 0) {
        return;           // 조기 종료, 값 없음
    }
    printf("%d\n", n);
}   // 여기서 암묵적 return
```

> **미리보기 (8장):** 함수에서 여러 값을 효과적으로 반환하려면, 매개변수로 포인터를 전달한다. 함수는 이 포인터를 통해 호출자의 변수를 직접 수정할 수 있다.

### 2.3 함수 이름

함수 이름은 변수와 동일한 **식별자 규칙**을 따른다 (3장):

| 규칙 | 예시 |
|:----|:----|
| 문자 또는 `_`로 시작해야 함 | `factorial`, `_helper` |
| 문자, 숫자, `_`를 포함할 수 있음 | `calc2dArea` |
| C 키워드는 사용 불가 | ~~`return`~~, ~~`int`~~ |
| 대소문자를 구분함 | `factorial`과 `Factorial`은 서로 다름 |

**명명 규칙 (권장):**

| 규칙 | 스타일 | 예시 |
|:----|:------|:----|
| snake_case | `동사_명사` | `get_factorial`, `is_leap_year` |
| camelCase | `동사명사` | `getFactorial`, `isLeapYear` |
| PascalCase | `동사명사` | `GetDaysOfMonth`, `PrintMenu` |

> **팁:** **하나의** 명명 규칙을 선택하고 프로그램 전체에서 일관되게 사용하라. 함수가 하는 일을 설명하는 동사+명사 조합을 사용하라: `getFactorial`, `isLeapYear`, `printMenu`.

> **경고:** C에서는 **같은 이름의 함수를 두 개 정의할 수 없다** (오버로딩 불가). 프로그램 내에서 각 함수 이름은 고유해야 한다.

### 2.4 매개변수와 인수

**매개변수(parameter)**와 **인수(argument)**는 관련되어 있지만 서로 다른 개념이다:

| 용어 | 다른 이름 | 위치 | 역할 |
|:----|:---------|:----|:----|
| **매개변수** | 형식 매개변수 (formal parameter) | 함수 **정의**에서 | 전달받는 값을 저장하는 지역 변수 |
| **인수** | 실제 매개변수 (actual parameter) | 함수 **호출 지점**에서 | 함수에 전달하는 실제 값 또는 식 |

```c
//                  매개변수 (형식)
//                    ↓
int square(int x) {
    return x * x;
}

int main() {
    int result = square(5);   // 5는 인수 (실제)
    //                 ↑
    //               인수
    return 0;
}
```

**매개변수 선언 규칙:**

```c
// 올바름: 각 매개변수에 자체 타입이 필요
int add(int a, int b) {
    return a + b;
}

// 잘못됨: 변수 선언처럼 타입을 공유할 수 없음
// int add(int a, b) { ... }   ← 컴파일 오류!
```

**매개변수가 없는 함수:**

```c
// 두 형태 모두 "매개변수 없음"을 의미:
void printMenu(void) { ... }   // 명시적 void — 권장
void printMenu() { ... }       // 빈 괄호 — 정의에서는 유효
```

> **핵심:** **원형(선언)**에서 `void printMenu()`는 "매개변수가 지정되지 않음"(아무 것이나 전달 가능)을 의미하고, `void printMenu(void)`는 "매개변수가 0개"를 의미한다. 명확성을 위해 원형에서는 항상 `void`를 사용하라.

**값에 의한 전달 (pass by value):**

C에서 인수는 **값에 의해** 전달된다. 매개변수는 인수의 **복사본**을 받는다. 매개변수를 변경해도 원래 변수에는 **영향을 주지 않는다**.

```c
void tryToChange(int x) {
    x = 100;   // 지역 복사본만 수정
    printf("함수 내부: x = %d\n", x);   // 100
}

int main() {
    int num = 5;
    tryToChange(num);
    printf("함수 호출 후: num = %d\n", num);  // 여전히 5!
    return 0;
}
```

```
출력:
함수 내부: x = 100
함수 호출 후: num = 5
```

> **미리보기 (8장):** 호출자의 변수를 수정하려면 값 대신 **주소(포인터)**를 전달한다. 이를 "참조에 의한 전달" (포인터를 통한)이라고 한다.

### 2.5 함수 호출

함수를 실행하려면 함수 이름 뒤에 인수를 포함한 괄호를 작성한다.

```c
// 함수를 호출하고 반환값 사용
int result = factorial(5);              // 반환값을 저장
printf("8! = %d\n", factorial(8));      // 반환값을 식에서 직접 사용

// void 함수 호출
printLine();                            // 캡처할 반환값 없음
```

**함수 호출 규칙:**

1. 호출 지점 이전에 함수가 **정의되었거나 선언되어** 있어야 한다.
2. 인수의 개수와 타입이 매개변수와 일치해야 한다.
3. 반환값은 임의의 식에서 사용하거나, 변수에 대입하거나, 무시할 수 있다.

**다른 함수에서 함수 호출:**

```c
#include <stdio.h>

int isLeapYear(int year) {
    return (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
}

int GetDaysOfMonth(int year, int month) {
    if (month == 2) {
        return isLeapYear(year) ? 29 : 28;   // isLeapYear() 호출
    }
    switch (month) {
    case 4: case 6: case 9: case 11:
        return 30;
    default:
        return 31;
    }
}

int main() {
    int days = GetDaysOfMonth(2024, 2);       // GetDaysOfMonth() 호출
    printf("2024년 2월은 %d일\n", days);       // 29
    return 0;
}
```

### 2.6 함수 선언 (원형)

**함수 선언**(또는 **원형**, prototype)은 전체 정의가 나타나기 **전에** 함수의 인터페이스를 컴파일러에 알려준다.

**문법:**

```c
반환형 함수이름(매개변수_목록);   // ← 세미콜론, 본체 없음
```

**원형이 필요한 이유:**

C 컴파일러는 소스 코드를 **위에서 아래로** 읽는다. `main()`이 그 *아래에* 정의된 함수를 호출하면, 컴파일러는 아직 그 함수의 반환형이나 매개변수 타입을 모른다. 원형은 인터페이스를 미리 선언하여 이 문제를 해결한다.

```c
#include <stdio.h>

// ─── 원형 (선언) ───
int factorial(int n);         // 매개변수 이름 포함 (권장)
int isLeapYear(int);          // 매개변수 이름 생략 (유효)

int main() {
    printf("5! = %d\n", factorial(5));      // OK — 위에서 원형 확인됨
    printf("Leap? %d\n", isLeapYear(2024)); // OK — 위에서 원형 확인됨
    return 0;
}

// ─── 정의 ───
int factorial(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

int isLeapYear(int year) {
    return (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
}
```

**원형 규칙:**

| 규칙 | 예시 |
|:----|:----|
| 정의의 반환형 및 매개변수 타입과 일치해야 함 | `int factorial(int n);`은 `int factorial(int n) { ... }`과 일치 |
| 매개변수 **이름**은 원형에서 선택 사항 | `int add(int, int);`은 유효 |
| **세미콜론**으로 끝나야 함 | `int factorial(int n);` (`int factorial(int n) { }` 아님) |
| 일반적으로 **`main()` 앞** 또는 헤더 파일에 배치 | 가독성을 위한 관례 |

> **팁:** 매개변수 이름은 선택 사항이지만 원형에 항상 포함하라. 문서 역할을 한다: `double power(double base, int exponent);`가 `double power(double, int);`보다 훨씬 읽기 쉽다.

**원형 없이 — 어떤 문제가 발생하는가:**

```c
#include <stdio.h>

int main() {
    // factorial()이 아래에 정의되어 있고 위에 원형이 없는 경우:
    // 컴파일러가 int 반환형으로 가정하거나 (C89) 오류를 발생시킴 (C99+)
    printf("5! = %d\n", factorial(5));   // 경고 또는 오류!
    return 0;
}

int factorial(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) result *= i;
    return result;
}
```

> **경고:** C99 이후에서는 사전 선언이나 정의 없이 함수를 호출하면 **오류**가 발생한다. 항상 원형을 제공하거나 사용 전에 함수를 정의하라.

## 3. 지역 변수와 전역 변수

### 3.1 지역 변수

**지역 변수**는 블록 `{}` 내부에서 선언된다 — 일반적으로 함수 본체 안에서. 해당 블록이 실행되는 동안에만 존재한다.

```c
void funcA() {
    int x = 10;   // funcA의 지역 변수
    printf("funcA: x = %d\n", x);
}

void funcB() {
    int x = 20;   // funcB의 지역 변수 — funcA의 x와 독립적
    printf("funcB: x = %d\n", x);
}

int main() {
    funcA();   // 출력: funcA: x = 10
    funcB();   // 출력: funcB: x = 20
    // printf("%d", x);   // 오류 — 여기서 x에 접근 불가
    return 0;
}
```

**지역 변수의 특성:**

| 속성 | 설명 |
|:----|:----|
| 유효 범위 (scope) | 선언 위치부터 해당 `{}` 블록의 끝까지 |
| 수명 (lifetime) | 블록에 진입할 때 생성되고, 블록을 벗어나면 소멸 |
| 초기값 | 명시적으로 초기화하지 않으면 **미정의**(쓰레기 값) |
| 독립성 | 서로 다른 함수에서 같은 변수 이름을 충돌 없이 사용 가능 |

```c
int main() {
    for (int i = 0; i < 3; i++) {
        int temp = i * 10;   // 매 반복마다 생성되고, 반복 끝에 소멸
        printf("%d ", temp);
    }
    // printf("%d", temp);  // 오류 — temp는 여기에 존재하지 않음
    return 0;
}
```

> **경고:** 지역 변수는 자동으로 0으로 초기화되지 **않는다**. 사용 전에 항상 초기화하라. 그렇지 않으면 예측 불가능한 쓰레기 값이 나온다.

### 3.2 전역 변수

**전역 변수**는 **모든 함수 바깥**에서 선언되며, 일반적으로 파일 상단에 위치한다. 선언 지점부터 파일 끝까지 접근 가능하다.

```c
#include <stdio.h>

int count = 0;   // 전역 변수 — 이 줄 아래 어디서든 접근 가능

void increment() {
    count++;   // 전역 변수를 수정
}

void printCount() {
    printf("count = %d\n", count);
}

int main() {
    increment();
    increment();
    increment();
    printCount();   // count = 3
    return 0;
}
```

**전역 변수의 특성:**

| 속성 | 설명 |
|:----|:----|
| 유효 범위 (scope) | 선언 지점부터 소스 파일의 끝까지 |
| 수명 (lifetime) | 프로그램 전체 실행 기간 (프로그램 시작 시 생성, 종료 시 소멸) |
| 초기값 | **자동으로 `0`으로 초기화됨** (지역 변수와 다름) |
| 접근성 | 선언 이후에 정의된 모든 함수에서 접근 가능 |

```c
#include <stdio.h>

int g;   // 전역 — 자동으로 0으로 초기화됨

int main() {
    printf("g = %d\n", g);   // g = 0 (보장됨)

    int local;
    // printf("local = %d\n", local);  // 위험 — 미초기화, 쓰레기 값
    return 0;
}
```

> **경고:** 전역 변수를 남용하지 마라. 어떤 함수가 전역 변수를 수정하는지 추적하기 어려워져 **숨겨진 의존성**과 디버그하기 힘든 버그가 발생한다. 값을 매개변수로 전달하는 것을 선호하라.

### 3.3 유효 범위 규칙

지역 변수와 전역 변수의 이름이 같으면, 해당 유효 범위 내에서 **지역 변수가 우선한다**. 이를 **섀도잉(shadowing)**이라 한다.

```c
#include <stdio.h>

int x = 100;   // 전역

void demo() {
    int x = 50;   // 지역 — 전역 x를 가림
    printf("demo() 내부: x = %d\n", x);       // 50 (지역)
}

int main() {
    printf("demo() 전: x = %d\n", x);          // 100 (전역)
    demo();
    printf("demo() 후: x = %d\n", x);          // 100 (전역, 변경 없음)

    {
        int x = 999;   // 블록 지역 — 전역 x를 가림
        printf("블록 내부: x = %d\n", x);       // 999
    }
    printf("블록 후: x = %d\n", x);             // 100 (전역)

    return 0;
}
```

```
출력:
demo() 전: x = 100
demo() 내부: x = 50
demo() 후: x = 100
블록 내부: x = 999
블록 후: x = 100
```

**유효 범위 우선순위:**

```
동일한 이름이 여러 수준에 존재할 때:
  내부 블록 변수  >  외부 블록 변수  >  전역 변수
  (가장 안쪽 범위가 우선)
```

| 범위 수준 | 변수 | 접근 가능한 곳 |
|:---------|:----|:-------------|
| 블록 `{}` | 블록 내부에서 선언 | 해당 블록 내에서만 |
| 함수 | 함수 상단에서 선언 | 함수 전체에서 |
| 파일 (전역) | 모든 함수 바깥에서 선언 | 선언 지점부터 파일 끝까지 |

> **팁:** 지역 변수와 전역 변수에 같은 이름을 사용하지 마라. 섀도잉은 코드를 혼란스럽게 만들고 흔한 버그의 원인이 된다. 구별되고 설명적인 이름을 사용하라.

## 4. 함수 호출 오버헤드

모든 함수 호출에는 **호출 스택**이 관리하는 일련의 배후 작업이 수반된다:

```
함수 호출 메커니즘
─────────────────────────────────────────
1. 복귀 주소 저장 (돌아올 위치)
2. CPU 레지스터 값 저장
3. 인수를 스택에 push
4. 함수 코드로 점프
5. 함수 본체 실행
6. 반환값을 레지스터에 배치
7. 스택 pop (레지스터, 인수 복원)
8. 복귀 주소로 점프
─────────────────────────────────────────
```

이 과정을 **함수 호출 오버헤드**라 한다. 상당한 계산을 수행하는 큰 함수의 경우 오버헤드는 무시할 수 있다. 그러나 **매우 작은 함수**(예: 한 줄짜리 계산)의 경우, 실제 작업에 비해 오버헤드가 상대적으로 클 수 있다.

```c
// 작은 함수 — 수행하는 작업에 비해 오버헤드가 상대적으로 큼
int square(int x) {
    return x * x;
}

// 촘촘한 반복문에서 호출 — 오버헤드 누적
for (int i = 0; i < 1000000; i++) {
    result += square(i);   // 함수 호출 오버헤드 × 1,000,000
}
```

> **핵심:** C++에서는 이 오버헤드를 줄이기 위해 `inline` 함수를 도입했다: 컴파일러가 함수 본체를 호출 지점에 직접 대입하여 호출/복귀 메커니즘을 제거한다. C99에서도 `inline` 키워드를 지원하지만, C++과 동작이 약간 다르다.

> **팁:** 오버헤드를 줄이기 위해 함수 사용을 피하지 마라. 함수의 가독성, 유지보수성, 재사용성 이점은 거의 모든 경우에서 작은 성능 비용을 훨씬 능가한다. 프로파일링으로 실제 병목이 확인된 경우에만 최적화하라.

## 5. 재귀 함수 (소개)

**재귀 함수**는 **자기 자신을 호출**하는 함수이다. 모든 재귀 함수에는 다음이 필요하다:

1. **기저 조건 (base case)** — 재귀를 멈추는 조건
2. **재귀 조건 (recursive case)** — 더 작거나 간단한 입력으로 자기 자신을 호출

```c
#include <stdio.h>

// 팩토리얼: n! = n × (n-1) × ... × 2 × 1
int factorial(int n) {
    if (n <= 1) {         // 기저 조건
        return 1;
    }
    return n * factorial(n - 1);   // 재귀 조건
}

int main() {
    printf("5! = %d\n", factorial(5));   // 120
    return 0;
}
```

**`factorial(4)` 실행 추적:**

```
factorial(4)
  → 4 * factorial(3)
       → 3 * factorial(2)
            → 2 * factorial(1)
                 → returns 1      (기저 조건)
            → returns 2 * 1 = 2
       → returns 3 * 2 = 6
  → returns 4 * 6 = 24
```

**다른 예제 — 양의 정수의 자릿수 세기:**

```c
int countDigits(int n) {
    if (n < 10) {         // 기저 조건: 한 자리 수
        return 1;
    }
    return 1 + countDigits(n / 10);   // 재귀 조건
}
// countDigits(9128) = 1 + countDigits(912)
//                   = 1 + 1 + countDigits(91)
//                   = 1 + 1 + 1 + countDigits(9)
//                   = 1 + 1 + 1 + 1 = 4
```

> **경고:** 적절한 기저 조건이 없는 재귀 함수는 무한히 자기 자신을 호출하여 **스택 오버플로**(호출 스택의 메모리가 고갈)를 일으킨다. 기저 조건에 도달할 수 있는지 항상 확인하라.

> **핵심:** 재귀로 풀 수 있는 모든 문제는 반복문(iteration)으로도 풀 수 있다. 재귀는 자기 유사적(self-similar)인 문제(팩토리얼, 피보나치, 트리 순회)에 우아하지만, 반복적 해법이 함수 호출 오버헤드가 낮아 종종 더 효율적이다.

| 함수 원형 | `반환형 이름(매개변수);` — 정의 전 호출 허용; 매개변수 이름 선택 사항 |
| 지역 변수 | `{}` 내부에서 선언; 블록 실행 중에만 존재; 자동 초기화 안 됨 |
| 전역 변수 | 모든 함수 바깥에서 선언; 파일 전체에서 접근 가능; 자동으로 `0`으로 초기화 |
| 유효 범위 규칙 | 안쪽 범위가 바깥쪽 범위를 가림; 같은 이름일 때 지역 > 전역 |
| 함수 호출 오버헤드 | 레지스터 저장/복원, 스택 push/pop; 작은 함수는 오버헤드가 상대적으로 큼 |
| 재귀 함수 | 자기 자신을 호출; 기저 조건 + 재귀 조건 필요; 스택 오버플로 위험 |
| 다중 반환값 | 직접 불가; 포인터 사용 (8장 미리보기) |
