## 1. 첫 번째 C 프로그램

### 1.1 프로그램 구조

모든 C 프로그램은 기본적인 구조를 따른다. 가장 간단한 완전한 프로그램을 한 줄씩 주석으로 설명하면 다음과 같다:

```c
// first.c
#include <stdio.h>    // 입출력 라이브러리 헤더

int main(void)        // 진입점 함수
{
    printf("First C Program\n");  // 콘솔 출력
    return 0;         // OS에 종료 코드 반환
}
```

```
C 프로그램 구조
├── 전처리기 지시문       #include <stdio.h>
├── main 함수            int main(void) { ... }
│   ├── 실행문           printf("...\n");
│   └── 반환문           return 0;
└── 기타 함수            (이후 장에서 학습)
```

| 구성 요소 | 역할 |
|:---------|:-----|
| `#include <stdio.h>` | 표준 입출력 라이브러리를 포함하여 `printf`와 `scanf`를 사용할 수 있게 함 |
| `int main(void)` | 진입점 정의; OS가 가장 먼저 호출하는 함수 |
| `{ ... }` | 중괄호로 함수 본문을 구분 |
| `printf("...\n");` | 문자열을 콘솔에 출력 |
| `return 0;` | 정상 종료를 의미하는 종료 코드 0을 OS에 반환 |

> **핵심:** 함수 내부의 모든 문장은 세미콜론(`;`)으로 끝나야 한다. 전처리기 지시문(`#include`)에는 세미콜론을 붙이지 **않는다**.

### 1.2 주석

주석은 컴파일러가 무시하며, 사람이 읽기 위해 존재한다.

| 종류 | 문법 | 범위 |
|:----|:----|:----|
| 블록 주석 | `/* ... */` | 여러 줄에 걸칠 수 있음 |
| 줄 주석 | `// ...` | `//`부터 줄 끝까지 |

```c
/* 이것은 블록 주석이다.
   여러 줄에 걸칠 수 있다. */

// 이것은 줄 주석이다.

int x = 10;  // 코드 뒤에 붙이는 인라인 주석
```

**코드 주석 처리(comment out)** — 코드를 삭제하지 않고 임시로 비활성화:

```c
/* printf("이 줄은 비활성화됨.\n"); */

// printf("이 줄도 비활성화됨.\n");
```

> **경고:** 블록 주석은 **중첩할 수 없다**. `/* 외부 /* 내부 */ 아직 안쪽? */`는 컴파일 에러를 일으킨다. `*/`가 첫 번째 매칭에서 주석을 닫기 때문이다.

### 1.3 `main` 함수

`main` 함수는 모든 C 프로그램의 **진입점(entry point)**이다. OS가 컴파일된 프로그램을 실행하면, `main`부터 실행이 시작된다.

| 규칙 | 설명 |
|:----|:----|
| 반드시 하나만 | 모든 프로그램에는 `main` 함수가 하나만 있어야 한다 |
| 반환 타입 `int` | `main`은 정수 종료 코드를 OS에 반환한다 |
| 매개변수 `void` | `void`는 인자가 없음을 의미; 빈 괄호 `()`도 가능 |
| 대소문자 구분 | `main`은 유효; `Main`, `MAIN`, `mAiN`은 진입점이 **아님** |

```c
int main(void)    // ✅ 표준 형태
{
    return 0;
}
```

```c
int main()        // ✅ 유효 (빈 매개변수 목록)
{
    return 0;
}
```

> **핵심:** C는 **대소문자를 구분**한다. `main`과 `Main`은 서로 다른 식별자이다. 컴파일러가 진입점으로 인식하는 것은 소문자 `main`뿐이다.

### 1.4 종료 코드

`main`의 `return` 문은 **종료 코드(exit code, exit status)**를 운영체제에 전달한다.

| 종료 코드 | 의미 |
|:--------:|:----|
| `0` | 정상 종료 (성공) |
| 0이 아닌 값 | 비정상 종료 (오류) |

```c
int main(void)
{
    printf("Hello\n");
    return 0;   // OS에 알림: 프로그램이 정상적으로 완료됨
}
```

OS 또는 호출 스크립트는 이 종료 코드를 검사하여 프로그램의 성공/실패 여부를 판단할 수 있다.

> **팁:** C99 이후에서는 `main` 끝에 `return`을 생략하면 컴파일러가 자동으로 `0`을 반환한다. 그러나 명확성을 위해 `return 0;`을 명시적으로 작성하는 것이 좋다.

## 2. 입력과 출력

### 2.1 콘솔 입출력

C에는 입출력 키워드가 내장되어 있지 않다. 대신 **헤더 파일**에 선언된 **라이브러리 함수**를 통해 입출력을 수행한다.

```c
#include <stdio.h>   // stdio = standard input/output (표준 입출력)
```

| 용어 | 의미 |
|:----|:----|
| `#include` | 헤더 파일의 내용을 소스에 복사하는 전처리기 지시문 |
| `<stdio.h>` | 표준 입출력 헤더; `printf`, `scanf` 등을 제공 |
| 표준 입력 | 키보드 (기본값) |
| 표준 출력 | 콘솔/화면 (기본값) |
| 헤더 파일 | 컴파일러에 필요한 함수 선언을 담은 파일 (`.h`) |

```
┌──────────┐    stdin     ┌─────────┐    stdout    ┌──────────┐
│   키보드   │ ──────────▶  │  프로그램 │ ──────────▶  │   화면    │
└──────────┘   (scanf)    └─────────┘   (printf)   └──────────┘
```

> **핵심:** `#include <stdio.h>`가 없으면 컴파일러가 `printf`와 `scanf`의 선언을 알 수 없어 에러(또는 표준에 따라 경고)가 발생한다.

### 2.2 `printf`

`printf`(print formatted)는 형식화된 출력을 콘솔에 출력한다.

**기본 문법:**

```c
printf("형식 문자열", 값1, 값2, ...);
```

**형식 문자열**에 포함할 수 있는 요소:
- 일반 텍스트 — 그대로 출력
- **형식 지정자** — 대응하는 값으로 대치되는 자리표시자
- **이스케이프 시퀀스** — `\n` 등의 특수 문자

#### 형식 지정자

| 지정자 | 타입 | 예시 |
|:-----:|:----|:----|
| `%d` | `int` (10진 정수) | `printf("%d", 42);` → `42` |
| `%f` | `float`/`double` | `printf("%f", 3.14);` → `3.140000` |
| `%c` | `char` (문자) | `printf("%c", 'A');` → `A` |
| `%s` | 문자열 | `printf("%s", "hi");` → `hi` |
| `%x` | `int` (16진수) | `printf("%x", 255);` → `ff` |
| `%e` | `float`/`double` (지수 표기) | `printf("%e", 0.001);` → `1.000000e-03` |

```c
int age = 20;
double height = 175.5;

printf("나이: %d\n", age);           // 나이: 20
printf("키: %f cm\n", height);       // 키: 175.500000 cm
printf("키: %.1f cm\n", height);     // 키: 175.5 cm
```

#### 줄바꿈과 커서 동작

`printf`는 자동으로 다음 줄로 이동하지 **않는다**. 커서는 출력된 텍스트 끝에 그대로 남는다. `\n`을 사용하여 줄바꿈을 삽입해야 한다.

```c
printf("Hello");
printf("World\n");
// 출력: HelloWorld   (공백이나 줄바꿈 없이 이어서 출력)
```

```c
printf("1번 줄\n");
printf("2번 줄\n");
// 출력:
// 1번 줄
// 2번 줄
```

> **팁:** 여러 `printf` 호출은 커서가 마지막으로 위치한 곳부터 이어서 출력한다. `printf`는 현재 커서 위치에 텍스트를 덧붙인다고 생각하면 된다.

### 2.3 `scanf`

`scanf`(scan formatted)는 키보드로부터 형식화된 입력을 읽는다.

**기본 문법:**

```c
scanf("형식 문자열", &변수1, &변수2, ...);
```

`&`(**주소 연산자**)는 각 변수 앞에 반드시 붙여야 한다. `scanf`는 입력값을 저장할 **메모리 주소**가 필요하다.

```c
int num;
printf("숫자를 입력하세요: ");
scanf("%d", &num);            // 키보드에서 정수를 읽음
printf("입력한 값: %d\n", num);
```

#### 여러 값 읽기

```c
int a, b;
scanf("%d %d", &a, &b);  // 사용자 입력: 10 20
// a = 10, b = 20
```

기본적으로 `scanf`는 공백(스페이스, 탭, 줄바꿈)을 값 사이의 구분자로 사용한다.

```c
int day, month, year;
scanf("%d/%d/%d", &day, &month, &year);  // 사용자 입력: 16/03/2026
// day = 16, month = 3, year = 2026
```

> **경고:** `scanf`에서 `&`를 빠뜨리는 것은 초보자가 흔히 하는 실수이다. `&`가 없으면 변수의 **주소**가 아닌 **값**(대부분 쓰레기 값)이 전달되어 정의되지 않은 동작이나 프로그램 충돌이 발생한다.

#### `scanf`의 형식 지정자

| 지정자 | 타입 | 예시 |
|:-----:|:----|:----|
| `%d` | `int` | `scanf("%d", &n);` |
| `%f` | `float` | `scanf("%f", &f);` |
| `%lf` | `double` | `scanf("%lf", &d);` |
| `%c` | `char` | `scanf("%c", &ch);` |
| `%s` | 문자열 (char 배열) | `scanf("%s", str);` (배열은 `&` 불필요) |

> **핵심:** `double`의 경우, `printf`는 `%f`를 사용하지만 `scanf`는 `%lf`를 사용한다. 이것은 흔한 버그의 원인이다.

### 2.4 입출력에서의 변수 활용

변수는 `printf`나 `scanf`에서 사용하기 전에 **선언**해야 한다.

**선언** — 컴파일러에게 변수의 타입과 이름을 알려줌:

```c
int count;       // 정수형 변수 선언
double price;    // 실수형 변수 선언
```

**대입** — 변수에 값을 저장:

```c
count = 5;
price = 19.99;
```

**선언 + 초기화** — 선언과 대입을 하나의 문장으로:

```c
int count = 5;
double price = 19.99;
```

**`printf`와 `scanf`에서 변수 사용:**

```c
#include <stdio.h>

int main(void)
{
    int quantity;
    double unit_price;

    printf("수량과 단가를 입력하세요: ");
    scanf("%d %lf", &quantity, &unit_price);   // 입력

    double total = quantity * unit_price;

    printf("수량: %d\n", quantity);              // 출력
    printf("단가: %.2f\n", unit_price);
    printf("합계: %.2f\n", total);

    return 0;
}
```

> **팁:** 변수를 사용하기 전에 항상 초기화할 것. 초기화되지 않은 변수에는 쓰레기 값이 들어 있어 예측 불가능한 결과를 초래한다.

## 3. C 문장

C 프로그램은 **문장(statement)**으로 구성된다. 문장은 컴퓨터가 순서대로 실행하는 명령이며, 모든 문장은 세미콜론(`;`)으로 끝난다.

| 문장 유형 | 역할 | 예시 | 다루는 장 |
|:---------|:----|:----|:---------|
| 선언문 | 변수를 생성 | `int x;` | 2장 |
| 입력문 | 키보드에서 데이터를 읽음 | `scanf("%d", &x);` | 2장 |
| 출력문 | 화면에 데이터를 표시 | `printf("%d", x);` | 2장 |
| 대입문 | 값을 저장 | `x = 10;` | 2장 |
| 조건문 | 조건에 따라 코드를 실행 | `if (x > 0) { ... }` | 5장 |
| 반복문 | 코드를 반복 실행 | `for (i = 0; i < 10; i++) { ... }` | 5장 |
| 함수 호출 | 다른 함수를 호출 | `result = add(a, b);` | 6장 |

```c
int main(void)
{
    int a, b, sum;            // 선언문
    scanf("%d %d", &a, &b);  // 입력문
    sum = a + b;              // 대입문
    printf("%d\n", sum);      // 출력문
    return 0;                 // 반환문
}
```

문장은 **순차적으로** 실행된다 — 위에서 아래로, 하나씩. 제어문(조건문, 반복문)이 흐름을 변경하지 않는 한 순서대로 진행된다.

> **참고:** 조건문(`if`, `switch`)과 반복문(`for`, `while`, `do-while`)은 여기서 간략히 소개하며, 자세한 내용은 5장에서 다룬다.

## 요약

| 개념 | 핵심 정리 |
|:----|:---------|
| 프로그램 구조 | `#include` → `main` 함수 → 실행문 → `return` |
| 주석 | `/* 블록 */` 과 `// 줄`; 블록 주석은 중첩 불가 |
| `main` 함수 | 진입점; 프로그램당 하나; `int` 반환; 대소문자 구분 |
| 종료 코드 | `return 0` = 성공; 0이 아닌 값 = 오류; C99+에서 생략 가능 |
| `#include <stdio.h>` | `printf`와 `scanf` 사용에 필수; 표준 입출력 헤더 |
| `printf` | 형식화된 출력; `%d`, `%f`, `%c`, `%s`, `%x`, `%e`; 커서는 마지막 문자 뒤에 위치 |
| `\n` | 줄바꿈 이스케이프; `printf`는 자동 줄바꿈하지 않음 |
| `scanf` | 형식화된 입력; `&`(주소 연산자) 필수; `double`은 `%lf` 사용 |
| 변수 | 사용 전 선언 필수; 초기화하여 쓰레기 값 방지 |
| 문장 | 선언문, 입력문, 출력문, 대입문, 조건문, 반복문 — 모두 `;`로 종료 |
