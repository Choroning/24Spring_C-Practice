## 1. 포인터 기초

### 1.1 포인터 개념

메모리는 **연속된 바이트**의 나열이며, 각 바이트는 고유한 **주소(address)**를 갖는다. **포인터(pointer)**란 주소 그 자체이고, **포인터 변수**란 주소를 저장하는 변수이다.

```
메모리 구조 (간략화)
주소          값
┌──────────┬──────────┐
│ 0x1000   │   10     │  ← int a = 10;
├──────────┼──────────┤
│ 0x1004   │   20     │  ← int b = 20;
├──────────┼──────────┤
│ 0x1008   │ 0x1000   │  ← int *p = &a;  (p에 a의 주소 저장)
└──────────┴──────────┘
```

| 아키텍처 | 포인터 크기 | 주소 범위 |
|:---------|:-----------|:---------|
| 32비트 | 4바이트 | `0x00000000` ~ `0xFFFFFFFF` |
| 64비트 | 8바이트 | `0x0000000000000000` ~ `0xFFFFFFFFFFFFFFFF` |

> **핵심:** 포인터는 변수의 이름을 모르더라도 **주소를 통해** 해당 변수에 접근할 수 있게 해 준다. 이것이 포인터의 핵심 능력이다.

### 1.2 포인터 선언

**문법:**

```c
자료형 *포인터이름;
```

`자료형`은 포인터가 가리킬 변수의 타입이다. `*`는 이 변수가 포인터임을 나타낸다.

```c
int *p;      // int를 가리키는 포인터
double *q;   // double을 가리키는 포인터
char *r;     // char를 가리키는 포인터
```

**`*`의 위치는 상관없다 — 세 가지 형태 모두 동일:**

```c
int* p;    // OK
int *p;    // OK (권장)
int * p;   // OK
```

> **경고:** `int *p` 스타일을 권장한다. `int* p, q;`라고 쓰면 `p`만 포인터이고 `q`는 일반 `int`이다. `int *p, *q;`라고 써야 둘 다 포인터임이 명확하다.

```c
int* p, q;    // p는 int*, q는 int   (혼동 유발!)
int *p, *q;   // p, q 모두 int*      (명확)
```

**포인터 크기는 가리키는 타입에 관계없이 동일:**

```c
int *p;
double *q;
char *r;

printf("%zu\n", sizeof(p));  // 8 (64비트 시스템)
printf("%zu\n", sizeof(q));  // 8 (64비트 시스템)
printf("%zu\n", sizeof(r));  // 8 (64비트 시스템)
```

모든 포인터는 32비트에서 **4바이트**, 64비트에서 **8바이트**를 차지한다. 데이터가 아니라 주소를 저장하기 때문이다.

### 1.3 포인터 초기화

**`&`(주소 연산자)**는 변수의 메모리 주소를 반환한다.

```c
int a = 10;
int *p = &a;   // p에 a의 주소를 저장

printf("a의 주소: %p\n", (void *)&a);   // 예: 0x7ffee3c0
printf("p의 값:   %p\n", (void *)p);    // 같은 주소
```

**`&` 연산자의 제약:** `&`는 **변수(lvalue)**에만 사용할 수 있다. 다음에는 사용할 수 없다:

```c
&10        // 오류: 리터럴 (주소 없음)
&(a + 1)   // 오류: 수식 결과 (임시값, 주소 없음)
#define MAX 100
&MAX       // 오류: 매크로 (전처리 시 치환됨, 주소 없음)
```

**NULL 포인터:** 아무것도 가리키지 않는 포인터.

```c
int *q = NULL;   // 명시적으로 아무것도 가리키지 않음
int *r = 0;      // NULL과 동일
```

> **팁:** 포인터는 항상 초기화하라. 초기화되지 않은 포인터는 쓰레기 주소를 갖고 있으며, 역참조하면 **정의되지 않은 동작**(크래시, 데이터 손상 등)이 발생한다.

**형식 지정자 `%p`**는 주소를 16진수로 출력한다:

```c
int x = 42;
int *p = &x;
printf("p = %p\n", (void *)p);   // p = 0x7ffee4b8 (예시)
```

### 1.4 역참조 연산자 `*`

**역참조(indirection)** 연산자 `*`는 포인터가 가리키는 주소에 저장된 값에 접근한다. **읽기**와 **쓰기** 모두 가능하다.

```c
int a = 10;
int *p = &a;

printf("%d\n", *p);   // 10 (읽기: p가 가리키는 주소의 값)

*p = 20;               // 쓰기: p가 가리키는 주소의 값을 변경
printf("%d\n", a);     // 20 (p를 통해 a가 수정됨)
```

`p`가 `a`를 가리킬 때, `*p`와 `a`는 **같은 메모리 위치의 다른 이름**이다:

```
   p ──────► ┌───────┐
             │  20   │  ← *p와 a는 같은 위치를 가리킴
             └───────┘
             주소: 0x1000
```

**`*` 연산자의 제약:** `*`는 **포인터 변수**에만 사용할 수 있다. 다음에는 사용할 수 없다:

```c
int a = 10;
*a         // 오류: a는 포인터가 아님
*(a + 1)   // 오류: (a + 1)은 포인터가 아님
```

> **핵심:** 포인터 문맥에서 `*` 기호는 두 가지 의미를 갖는다: (1) 선언문 `int *p;`에서는 "p는 포인터"를 의미하고, (2) 수식 `*p`에서는 "p를 역참조"(p가 가리키는 주소의 값에 접근)를 의미한다.

### 1.5 포인터의 용도

포인터를 왜 사용하는가? 교재에서는 두 가지 핵심 상황을 강조한다:

| 상황 | 설명 |
|:----|:----|
| 함수에서 2개 이상의 값을 반환해야 할 때 | C 함수는 `return`으로 하나의 값만 반환 가능. 변수의 주소를 전달하면 함수 내에서 원본을 수정할 수 있다. |
| 함수에 배열을 전달할 때 | 배열은 전달 시 포인터로 변환(decay)된다. 포인터를 사용하면 복사 없이 효율적으로 배열에 접근할 수 있다. |

```c
// 예제: 포인터를 통한 다중 값 반환
void divide(int a, int b, int *quotient, int *remainder) {
    *quotient = a / b;
    *remainder = a % b;
}

int main() {
    int q, r;
    divide(17, 5, &q, &r);
    printf("17 / 5 = %d 나머지 %d\n", q, r);  // 3 나머지 2
    return 0;
}
```

### 1.6 `const` 포인터

`const` 키워드는 포인터가 수정할 수 있는 대상을 제한한다.

| 선언 | 의미 | `*p` 변경 가능? | `p` 변경 가능? |
|:----|:----|:---------------|:--------------|
| `const int *p` | 상수 int를 가리키는 포인터 | 불가 | 가능 |
| `int *const p` | int를 가리키는 상수 포인터 | 가능 | 불가 |
| `const int *const p` | 상수 int를 가리키는 상수 포인터 | 불가 | 불가 |

```c
int a = 10, b = 20;

// 상수 int를 가리키는 포인터: *p1 수정 불가, p1 재대입 가능
const int *p1 = &a;
// *p1 = 30;    // 오류: p1을 통해 값을 수정할 수 없음
p1 = &b;        // OK: p1이 이제 b를 가리킴

// int를 가리키는 상수 포인터: *p2 수정 가능, p2 재대입 불가
int *const p2 = &a;
*p2 = 30;       // OK: a를 30으로 수정
// p2 = &b;     // 오류: p2가 가리키는 대상을 변경할 수 없음

// 상수 int를 가리키는 상수 포인터: 둘 다 불가
const int *const p3 = &a;
// *p3 = 40;    // 오류
// p3 = &b;     // 오류
```

> **팁:** 선언을 **오른쪽에서 왼쪽으로** 읽으면 이해하기 쉽다: `const int *p` = "p는 const인 int를 가리키는 포인터"(값이 상수). `int *const p` = "p는 int를 가리키는 const 포인터"(포인터 자체가 상수).

## 2. 배열과 포인터

### 2.1 포인터 연산

포인터에 정수를 더하거나 빼면, 컴파일러는 1바이트가 아니라 **가리키는 타입의 크기**만큼 이동한다.

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;    // p는 arr[0]을 가리킴
```

```
p       p+1     p+2     p+3     p+4
│       │       │       │       │
▼       ▼       ▼       ▼       ▼
┌──────┬──────┬──────┬──────┬──────┐
│  10  │  20  │  30  │  40  │  50  │
└──────┴──────┴──────┴──────┴──────┘
0x1000  0x1004  0x1008  0x100C  0x1010
        (+4B)   (+4B)   (+4B)   (+4B)
```

| 포인터 타입 | `p + 1` 이동량 |
|:-----------|:--------------|
| `int *` | 4바이트 (`sizeof(int)`) |
| `double *` | 8바이트 (`sizeof(double)`) |
| `char *` | 1바이트 (`sizeof(char)`) |

**지원되는 연산:**

| 연산 | 의미 |
|:----|:----|
| `p + n` | n개 요소만큼 앞으로 이동 |
| `p - n` | n개 요소만큼 뒤로 이동 |
| `p++` | 다음 요소로 이동 |
| `p--` | 이전 요소로 이동 |
| `p1 - p2` | 두 포인터 사이의 요소 수 (바이트 수가 아님) |

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

printf("%d\n", *(p + 2));   // 30 (세 번째 요소)

p++;
printf("%d\n", *p);         // 20 (두 번째 요소)

int *q = &arr[4];
printf("%ld\n", q - p);     // 3 (요소 단위 거리, 바이트가 아님)
```

> **경고:** 포인터 연산은 배열 범위 내(또는 배열 끝의 바로 다음 위치까지)에서만 유효하다. 범위를 벗어나면 **정의되지 않은 동작**이 발생한다.

### 2.2 포인터를 배열처럼 사용

포인터에 **첨자 연산자 `[]`**를 사용할 수 있다. `p[i]`는 `*(p + i)`와 동일하다.

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

printf("%d\n", p[0]);       // 10  (*(p + 0)과 동일)
printf("%d\n", p[2]);       // 30  (*(p + 2)과 동일)
printf("%d\n", *(p + 4));   // 50  (p[4]와 동일)
```

| 표기법 | 동치 | 의미 |
|:------|:----|:----|
| `p[i]` | `*(p + i)` | p에서 i번째 요소에 접근 |
| `&p[i]` | `p + i` | p에서 i번째 요소의 주소 |

### 2.3 배열을 포인터처럼 사용

**배열 이름**은 첫 번째 요소의 주소와 같다. 즉, **포인터 상수(pointer constant)**처럼 동작한다.

```c
int arr[5] = {10, 20, 30, 40, 50};

printf("%p\n", (void *)arr);       // arr[0]의 주소
printf("%p\n", (void *)&arr[0]);   // 같은 주소

printf("%d\n", *arr);         // 10  (arr[0]과 동일)
printf("%d\n", *(arr + 2));   // 30  (arr[2]과 동일)
```

**핵심 등가 관계:**

| 배열 표기법 | 포인터 표기법 |
|:----------|:-----------|
| `arr` | `&arr[0]` |
| `arr[i]` | `*(arr + i)` |
| `&arr[i]` | `arr + i` |

> **경고:** 배열 이름은 **상수**이므로 수정할 수 없다. 다음은 **불가능**하다:
> ```c
> arr++;          // 오류: 상수를 증가시킬 수 없음
> arr = other;    // 오류: 배열 이름에 재대입할 수 없음
> ```

### 2.4 배열과 포인터 비교

| 특성 | 배열 (`int arr[5]`) | 포인터 (`int *p`) |
|:----|:-------------------|:-----------------|
| 주소 | **고정** (변경 불가) | **가변** (재대입 가능) |
| `sizeof` | 배열 전체 크기 (`5 * 4 = 20`) | 포인터 크기 (`4` 또는 `8`) |
| 메모리 할당 | 선언 시 할당됨 | 기존 메모리를 가리켜야 함 |
| `++` 연산자 | 불가 (`arr++`는 오류) | 가능 (`p++`로 다음 요소 이동) |
| 정체성 | `arr`은 배열 그 자체 | `p`는 독립된 변수 |

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

printf("%zu\n", sizeof(arr));   // 20 (5 * sizeof(int))
printf("%zu\n", sizeof(p));     // 8  (64비트에서 포인터 크기)

// arr++;   // 오류
p++;        // OK: p는 이제 arr[1]을 가리킴
```

> **핵심:** `arr`과 `p`는 많은 수식에서 같은 방식으로 사용할 수 있지만(`arr[i]` vs `p[i]`), 본질적으로 다르다. `arr`은 컴파일 시 결정되는 상수 주소이고, `p`는 실행 시 주소를 저장하는 변수이다.

## 3. 함수와 포인터

### 3.1 값에 의한 전달 vs 주소에 의한 전달

| 방식 | 메커니즘 | 원본 수정 가능? |
|:----|:--------|:-------------|
| 값에 의한 전달 (call by value) | 함수가 인수의 **복사본**을 받음 | 불가 |
| 주소에 의한 전달 (call by reference) | 함수가 포인터를 통해 **주소**를 받음 | 가능 |

```c
void by_value(int x) {
    x = 100;   // 지역 복사본만 수정
}

void by_reference(int *x) {
    *x = 100;  // 원본 변수를 수정
}

int main() {
    int a = 10;

    by_value(a);
    printf("%d\n", a);     // 10 (변경 안 됨)

    by_reference(&a);
    printf("%d\n", a);     // 100 (수정됨)

    return 0;
}
```

```
값에 의한 전달                     주소에 의한 전달
┌──────────────┐                ┌──────────────┐
│ main         │                │ main         │
│  a = 10      │                │  a = 10 → 100│
└──────┬───────┘                └──────┬───────┘
       │ 복사                          │ 주소
       ▼                               ▼
┌──────────────┐                ┌──────────────┐
│ by_value     │                │ by_reference │
│  x = 10→100  │                │  x = &a      │
│  (지역 복사본)│                │  *x = 100    │
└──────────────┘                └──────────────┘
  a 변경 안 됨                     a 변경됨!
```

### 3.2 스왑(swap) 예제

주소에 의한 전달이 왜 필요한지를 보여 주는 대표적인 예제이다.

**잘못된 방법: 값에 의한 전달 (동작하지 않음)**

```c
void swap_wrong(int x, int y) {
    int temp = x;
    x = y;
    y = temp;
    // x, y는 지역 복사본 — 원본은 변경되지 않음
}

int main() {
    int a = 10, b = 20;
    swap_wrong(a, b);
    printf("a=%d, b=%d\n", a, b);   // a=10, b=20 (교환 안 됨!)
    return 0;
}
```

```
swap_wrong 호출 전:          swap_wrong 내부:             swap_wrong 후:
  main:  a=10, b=20          x=10→20, y=20→10           main: a=10, b=20
         │      │             (지역 복사본)               (변경 안 됨!)
         ▼      ▼
       복사    복사
         │      │
         ▼      ▼
       x=10   y=20
```

**올바른 방법: 주소에 의한 전달 (정상 동작)**

```c
void swap(int *x, int *y) {
    int temp = *x;
    *x = *y;
    *y = temp;
}

int main() {
    int a = 10, b = 20;
    swap(&a, &b);
    printf("a=%d, b=%d\n", a, b);   // a=20, b=10 (교환됨!)
    return 0;
}
```

```
swap 호출 전:                swap 내부:                   swap 후:
  main:  a=10, b=20          *x=10→20, *y=20→10         main: a=20, b=10
         │      │             (주소를 통해 원본 수정)      (교환됨!)
         ▼      ▼
       &a      &b
         │      │
         ▼      ▼
       x=&a   y=&b
       *x=10  *y=20
```

### 3.3 여러 값 반환

C 함수는 `return`으로 하나의 값만 반환할 수 있으므로, 포인터 매개변수를 사용하여 추가 값을 "반환"한다.

```c
void sec_to_hms(int sec, int *h, int *m, int *s) {
    *h = sec / 3600;
    *m = (sec % 3600) / 60;
    *s = sec % 60;
}

int main() {
    int hours, minutes, seconds;
    int total_sec = 3723;

    sec_to_hms(total_sec, &hours, &minutes, &seconds);
    printf("%d초 = %d시간 %d분 %d초\n", total_sec, hours, minutes, seconds);
    // 3723초 = 1시간 2분 3초

    return 0;
}
```

> **핵심:** 호출자는 변수의 **주소**를 전달한다. 함수는 해당 주소를 통해 `*`로 결과를 기록한다.

### 3.4 함수에 배열 전달

배열을 함수에 전달하면, 배열 이름은 첫 번째 요소에 대한 포인터로 **변환(decay)**된다. 함수는 배열의 복사본이 아니라 포인터를 받는다.

**두 가지 동일한 함수 시그니처:**

```c
void print_array(int *arr, int size);    // 포인터 문법
void print_array(int arr[], int size);   // 배열 문법 (동일)
```

두 표현은 동일하다. 함수 매개변수에서 `int arr[]`는 `int *arr`로 취급된다.

```c
void print_array(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);   // arr[i]는 *(arr + i)와 동일
    }
    printf("\n");
}

int main() {
    int data[5] = {10, 20, 30, 40, 50};
    print_array(data, 5);   // 배열 이름이 포인터로 변환됨
    return 0;
}
```

> **경고:** 함수 내부에서 `sizeof(arr)`는 배열 전체 크기가 아니라 **포인터 크기**(4 또는 8)를 반환한다. 그래서 크기를 별도의 매개변수로 전달해야 한다.

```c
void func(int arr[]) {
    printf("%zu\n", sizeof(arr));    // 8 (포인터 크기, 배열 크기가 아님!)
}

int main() {
    int arr[5] = {1, 2, 3, 4, 5};
    printf("%zu\n", sizeof(arr));    // 20 (배열 전체 크기)
    func(arr);
    return 0;
}
```

**포인터 매개변수를 통한 배열 요소 수정:**

```c
void double_elements(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        arr[i] *= 2;   // 원본 배열을 수정
    }
}

int main() {
    int nums[4] = {1, 2, 3, 4};
    double_elements(nums, 4);
    // nums는 이제 {2, 4, 6, 8}
    return 0;
}
```

> **팁:** 함수가 배열을 수정하지 **않아야** 한다면, 매개변수를 `const int *arr`로 선언하라. 컴파일러가 실수로 값을 변경하는 것을 방지해 준다.

## 요약

| 개념 | 핵심 정리 |
|:----|:---------|
| 포인터 | 주소 그 자체; 포인터 변수는 주소를 저장하는 변수 |
| 포인터 선언 | `자료형 *p;` — 자료형 = 가리킬 변수의 타입 |
| 포인터 크기 | 가리키는 타입에 관계없이 항상 4바이트(32비트) 또는 8바이트(64비트) |
| `&` (주소 연산자) | 변수의 주소를 반환; 리터럴, 수식, 매크로에는 사용 불가 |
| `*` (역참조 연산자) | 해당 주소의 값을 읽거나 씀; 포인터 변수에만 사용 가능 |
| NULL 포인터 | `int *p = NULL;` — 아무것도 가리키지 않음; 반드시 포인터를 초기화할 것 |
| `%p` | 주소를 16진수로 출력하는 형식 지정자 |
| `const` 포인터 | `const int *p`(값 상수), `int *const p`(포인터 상수), 또는 둘 다 |
| 포인터 연산 | `p + 1`은 1바이트가 아니라 `sizeof(*p)`바이트만큼 이동 |
| `p[i]` vs `*(p+i)` | 동일 — 첨자 표기와 포인터 연산은 서로 교환 가능 |
| 배열 이름 | 첫 번째 요소의 주소; 포인터 **상수**로 동작 (수정 불가) |
| 배열 vs 포인터 | 배열: 고정 주소, `sizeof` = 전체 크기; 포인터: 가변 주소, `sizeof` = 4/8 |
| 값에 의한 전달 | 함수가 복사본을 받음; 원본 변경 불가 |
| 주소에 의한 전달 | 함수가 포인터로 주소를 받음; 원본 수정 가능 |
| 스왑 함수 | 반드시 포인터 사용 (`int *x, int *y`) — 값에 의한 전달로는 교환 불가 |
| 다중 값 반환 | 함수에 주소를 전달; 함수가 `*`를 통해 값을 기록 |
| 함수에 배열 전달 | 배열이 포인터로 변환됨; 함수 내 `sizeof(arr)` = 포인터 크기; 크기를 별도로 전달 |
