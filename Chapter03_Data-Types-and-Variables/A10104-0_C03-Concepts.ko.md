## 1. 데이터 타입

**데이터 타입**은 값을 저장하는 데 필요한 메모리의 크기와 2진 표현을 결정한다. 컴파일러는 데이터 타입에 따라 몇 바이트를 할당하고, 저장된 비트를 어떻게 해석할지 결정한다.

```
데이터 타입
├── 기본 데이터 타입           ← 이번 장
│   ├── char (문자형)
│   ├── short, int, long, long long (정수형)
│   └── float, double, long double (실수형)
├── 파생 데이터 타입           ← 이후 장
│   ├── 배열
│   └── 포인터
└── 사용자 정의 타입          ← 이후 장
    ├── 구조체 (struct)
    ├── 공용체 (union)
    └── 열거체 (enum)
```

C에서 모든 변수는 **사용하기 전에** 타입을 선언해야 한다.

### 1.1 정수형

정수형은 소수점 없는 정수를 저장한다.

| 타입 | 크기 | 범위 (signed) | 형식 |
|:-----|:----:|:-------------|:----:|
| `char` | 1 B | -128 ~ 127 | `%d` |
| `short` | 2 B | -32,768 ~ 32,767 | `%hd` |
| `int` | 4 B | -2,147,483,648 ~ 2,147,483,647 | `%d` |
| `long` | 4 B | -2,147,483,648 ~ 2,147,483,647 | `%ld` |
| `long long` | 8 B | ±9.2 × 10¹⁸ | `%lld` |

C 표준은 `sizeof(short)` ≤ `sizeof(int)` ≤ `sizeof(long)`을 보장한다.

#### `signed`와 `unsigned`

기본적으로 정수형은 `signed`(음수 가능)이다. `unsigned`를 붙이면 음수 범위를 제거하고 양수 범위를 두 배로 확장한다.

```c
int a = -10;              // signed (기본) → 범위: -2B ~ +2B
unsigned int b = 100;     // unsigned → 범위: 0 ~ 4,294,967,295
```

| Signed 타입 | Unsigned 타입 | 크기 | Unsigned 범위 |
|:-----------|:------------|:----:|:------------|
| `char` | `unsigned char` | 1 B | 0 ~ 255 |
| `short` | `unsigned short` | 2 B | 0 ~ 65,535 |
| `int` | `unsigned int` | 4 B | 0 ~ 4,294,967,295 |
| `long` | `unsigned long` | 4 B | 0 ~ 4,294,967,295 |
| `long long` | `unsigned long long` | 8 B | 0 ~ 18,446,744,073,709,551,615 |

> **핵심:** `signed int`과 `int`은 같다. `unsigned`와 `unsigned int`도 같다.

#### 오버플로우와 언더플로우

모든 정수형에는 **고정된 범위**가 있다. 범위 밖의 값을 저장하면 **오버플로우**(최댓값 초과) 또는 **언더플로우**(최솟값 미만)가 발생한다. 값은 유효 범위 내로 순환(wrap around)된다.

```c
#include <limits.h>

int max = INT_MAX;       // 2,147,483,647
printf("%d\n", max + 1); // -2,147,483,648  (순환!)
```

```
          오버플로우
    ┌─── INT_MAX + 1 ───┐
    │                    ▼
... ─2 ─1  0  1  2 ... INT_MAX  INT_MIN ...
                         ▲                │
                         └── INT_MIN - 1 ─┘
                              언더플로우
```

```c
char n = 128;             // char 범위: -128~127 → 오버플로우!
printf("%d\n", n);        // -128  (순환)

unsigned char red = 300;  // unsigned char 범위: 0~255 → 오버플로우!
printf("%d\n", red);      // 44   (300 - 256 = 44)
```

> **팁:** `<limits.h>`를 포함하면 `INT_MAX`, `INT_MIN`, `SHRT_MAX` 등 타입 범위 상수를 사용할 수 있다.

### 1.2 실수형

실수형은 **부동소수점(floating point)** 방식으로 실수를 저장한다. 값은 `1.xxx × 2^n` 형태로 표현되며, 가수(xxx)가 정밀도를, 지수(n)가 범위를 결정한다.

| 타입 | 크기 | 정밀도 | 형식 | 범위 |
|:-----|:----:|:------|:----:|:-----|
| `float` | 4 B | 유효숫자 약 7자리 | `%f` | ±3.4 × 10³⁸ |
| `double` | 8 B | 유효숫자 약 15자리 | `%lf` | ±1.8 × 10³⁰⁸ |
| `long double` | 8+ B | 유효숫자 15자리 이상 | `%Lf` | ≥ `double` |

```c
float pi_f = 3.14f;       // 'f' 접미사로 float 리터럴 명시
double pi_d = 3.14159265; // 소수 리터럴은 기본적으로 double
```

> **경고:** `unsigned`는 실수형에 **적용할 수 없다**. `unsigned float`과 `unsigned double`은 잘못된 문법이다.

#### 정밀도(Precision)

**정밀도**란 실수형이 정확하게 저장할 수 있는 **유효숫자의 총 개수**이다. "소수점 이하 자릿수"가 아니다.

```c
float  f = 123456789.123456789f;
double d = 123456789.123456789;

printf("float:  %.15f\n", f);  // 123456792.000000000000000 (약 7자리 이후 틀림)
printf("double: %.15f\n", d);  // 123456789.123456791043282 (약 15자리 이후 틀림)
```

`float`는 유효숫자 약 7자리, `double`은 약 15자리이다. 대부분의 계산에서는 `double`을 사용한다.

#### 표현 오차

컴퓨터는 숫자를 **이진수(2진법)**로 저장한다. 일부 소수(예: 0.1)는 이진수로 정확히 표현할 수 없다. 마치 1/3을 10진수로 정확히 쓸 수 없는 것과 같다.

```c
double x = 0.1 + 0.2;
printf("%.20f\n", x);  // 0.30000000000000004441  (정확히 0.3이 아님!)

if (x == 0.3) { ... }  // ❌ false가 될 수 있다!
```

> **경고:** 실수끼리 `==`로 비교하지 말 것. 허용 오차(tolerance)를 사용한다:
> ```c
> #include <math.h>
> if (fabs(x - 0.3) < 1e-9) { /* 충분히 가까움 */ }
> ```

#### 실수형 오버플로우와 언더플로우

- **오버플로우** — 표현 가능한 최댓값 초과 → `INF`(무한대)로 설정.
- **언더플로우** — 표현 가능한 최솟값보다 작은 값 → 가수부를 줄이고 지수부를 늘려서 표현 → 결국 `0.0`으로 설정.

```c
#include <float.h>

float num = 3.40282e38f;    // FLT_MAX 근처 → 정상
num = 3.40282e40f;          // 오버플로우 → inf
num = 1.17549e-42f;         // 비정규화(정밀도 감소)
num = 1.17549e-46f;         // 언더플로우 → 0.0
```

### 1.3 문자형

`char`는 **ASCII** 인코딩을 사용하여 단일 문자를 저장한다 (1바이트). 내부적으로는 문자의 정수 코드를 저장한다.

```c
char letter = 'A';       // 내부적으로 65를 저장
printf("%c\n", letter);   // A  (문자로 출력)
printf("%d\n", letter);   // 65 (정수로 출력)
```

| 문자 | ASCII | 설명 |
|:----:|:-----:|:----|
| `'A'`~`'Z'` | 65~90 | 영문 대문자 |
| `'a'`~`'z'` | 97~122 | 영문 소문자 |
| `'0'`~`'9'` | 48~57 | 숫자 문자 |
| `' '` | 32 | 공백 |

ASCII 코드 0~31과 127은 **제어 문자**(출력 불가). 32~126은 **출력 가능 문자**이다.

`char`는 본질적으로 작은 정수이므로, 산술 연산이 가능하다:

```c
char ch = 'M';
char prev = ch - 1;    // 'L' (76)
char next = ch + 1;    // 'N' (78)
char lower = ch + 32;  // 'm' (109)
```

> **팁:** 대문자와 소문자의 차이는 항상 32이다 (`'a' - 'A' == 32`).

#### 이스케이프 시퀀스

이스케이프 시퀀스는 직접 입력할 수 없는 제어 문자나 특수 문자를 표현한다. 역슬래시(`\`)로 시작한다.

| 이스케이프 | 이름 | ASCII | 설명 |
|:--------:|:----|:-----:|:----|
| `\n` | 줄바꿈(newline) | 10 | 다음 줄로 이동 |
| `\t` | 탭(tab) | 9 | 수평 탭 |
| `\0` | 널(null) | 0 | 널 문자 (문자열 종료) |
| `\\` | 역슬래시 | 92 | 리터럴 `\` |
| `\"` | 큰따옴표 | 34 | 문자열 안에서 `"` 출력 |
| `\'` | 작은따옴표 | 39 | 문자 리터럴 안에서 `'` 출력 |

```c
printf("Hello\tWorld\n");    // Hello	World (탭 + 줄바꿈)
printf("She said \"Hi\"\n"); // She said "Hi"
printf("Path: C:\\Users\n"); // Path: C:\Users
```

`\x` 뒤에 16진수 ASCII 코드를 적어 문자를 표현할 수도 있다:

```c
char newline = '\xa';  // '\xa' == '\n' == ASCII 10
```

## 2. 2진 표현

### 2.1 진법

컴퓨터는 모든 데이터를 **이진수(2진법)**로 저장한다. 10진수, 2진수, 16진수 변환을 이해하는 것이 기본이다.

**10진수(base 10) → 각 자릿수는 10의 거듭제곱:**

```
45,238 = 4×10⁴ + 5×10³ + 2×10² + 3×10¹ + 8×10⁰
```

**2진수(base 2) → 각 자릿수는 2의 거듭제곱:**

```
11011₂ = 1×2⁴ + 1×2³ + 0×2² + 1×2¹ + 1×2⁰
       = 16   + 8    + 0    + 2    + 1    = 27₁₀
```

**16진수(base 16) → 숫자 0~9와 문자 A~F:**

| 10진 | 0 | 1 | ... | 9 | 10 | 11 | 12 | 13 | 14 | 15 |
|:---:|:-:|:-:|:---:|:-:|:--:|:--:|:--:|:--:|:--:|:--:|
| 16진 | 0 | 1 | ... | 9 | A  | B  | C  | D  | E  | F  |

```
0x3A0B = 3×16³ + 10×16² + 0×16¹ + 11×16⁰
       = 12288 + 2560   + 0     + 11     = 14,859₁₀
```

C에서 리터럴 접두사로 진법을 지정한다:

```c
int dec = 27;      // 10진수
int bin = 0b11011; // 2진수 (C99, 일부 컴파일러)
int oct = 033;     // 8진수 (접두사 0)
int hex = 0x1B;    // 16진수 (접두사 0x)
```

### 2.2 정수의 2진 표현 (2의 보수)

**컴퓨터는 음수를 어떻게 표현하는가?**

현대 시스템은 **2의 보수(two's complement)**를 사용한다. 최상위 비트(MSB)가 **부호 비트**이다: `0` = 양수, `1` = 음수.

**-n을 구하는 법:** n을 2진수로 쓰고 → 모든 비트를 반전 → 1을 더한다.

```
+7 (8비트):    0 0 0 0 0 1 1 1
비트 반전:      1 1 1 1 1 0 0 0
1을 더함:       1 1 1 1 1 0 0 1  → -7
```

**왜 2의 보수를 사용하는가?**
- 0의 표현이 하나뿐 (부호화 절대치 방식과 달리)
- 덧셈 회로가 양수/음수 구분 없이 동일하게 작동
- 하드웨어 구현이 간단

> **핵심:** 같은 비트 패턴이라도 타입에 따라 의미가 달라진다. `short a = -7`과 `unsigned short b = 65529`는 **동일한 비트**(`0xFFF9`)를 가지지만, 타입이 해석을 결정한다.

### 2.3 실수의 2진 표현 (IEEE 754)

실수는 **부호 비트 + 지수 + 가수**로 저장된다.

```
값 = (-1)^부호 × 1.가수 × 2^(지수 - 바이어스)
```

| 타입 | 부호 | 지수 | 가수 | 총 비트 |
|:-----|:----:|:----:|:----:|:------:|
| `float` (단정도) | 1비트 | 8비트 | 23비트 | 32비트 |
| `double` (배정도) | 1비트 | 11비트 | 52비트 | 64비트 |

- **가수 비트**가 많을수록 → 정밀도(유효숫자)가 높아진다.
- **지수 비트**가 많을수록 → 표현 가능한 범위가 넓어진다.

이것이 `float`의 유효숫자가 약 7자리(23비트 가수)이고, `double`이 약 15자리(52비트 가수)인 이유이다.

> **참고:** IEEE 754의 전체 인코딩 방식을 완벽히 이해할 필요는 없지만, 구조를 알면 `float`와 `double`의 정밀도와 범위 차이를 설명할 수 있다.

## 3. 변수

**변수**는 프로그램 실행 중 값을 변경할 수 있는 이름이 붙은 메모리 영역이다. **상수**는 값을 변경할 수 없는 데이터이다.

### 3.1 선언과 초기화

**선언**은 컴파일러에게 변수의 데이터 타입과 이름을 알려주는 것이다. **초기화**는 변수에 초기값을 지정하는 것이다.

```c
int x;                // 선언 — 값은 쓰레기 값(미정의)
int y = 10;           // 선언 + 초기화
x = 5;                // 대입

int a, b, c;          // 복수 선언
int m = 1, n = 2;    // 복수 선언 + 초기화
```

> **경고:** 초기화되지 않은 변수에는 **쓰레기 값**(이전에 메모리에 있던 임의의 값)이 들어 있다. 이를 사용하면 **정의되지 않은 동작**이 발생하며, 컴파일러가 경고를 출력한다.

초기값의 타입이 변수의 타입과 다르면 컴파일러가 자동 변환하지만 경고가 발생할 수 있다:

```c
float rate = 0.2;     // ⚠ 0.2는 double → float으로 변환 (정밀도 손실 가능)
int weight = 50.5;    // ⚠ 50.5는 double → int로 절삭 (데이터 손실)
```

### 3.2 식별자와 이름 규칙

**식별자(identifier)**는 변수, 함수 등에 프로그래머가 붙이는 이름이다.

| 규칙 | 유효한 예시 | 잘못된 예시 |
|:----|:---------|:---------|
| 영문자, 숫자, `_`만 사용 | `score1`, `tax_rate` | `annual-salary` (하이픈) |
| 첫 글자는 영문자 또는 `_` | `_amount`, `count` | `2019income` (숫자 시작) |
| 대소문자 구분 | `length` ≠ `Length` | — |
| 키워드 사용 불가 | — | `int`, `return`, `case` |
| 공백 포함 불가 | `taxRate` | `total amount` |

**이름 관례:**

```c
int usage2019;       // 숫자 사용 가능 (첫 글자만 아니면)
double _amount;      // _ 시작 가능
double tax_rate;     // snake_case: 단어를 _로 연결
double taxRate;      // camelCase: 연결되는 단어의 첫 글자를 대문자로
```

## 4. 상수와 리터럴

### 4.1 리터럴 상수

**리터럴 상수**는 소스 코드에서 직접 사용되는 값이다.

| 분류 | 구분 | 예 | 데이터 타입 |
|:-----|:----|:---|:---------|
| 문자형 | 일반 문자 | `'x'`, `'A'` | `int` |
| 문자형 | 이스케이프 시퀀스 | `'\n'`, `'\t'`, `'\xa'` | `int` |
| 정수형 | 10진수 | `-10`, `42` | `int` |
| 정수형 | 16진수 | `0xa`, `0xFF` | `int` |
| 정수형 | 8진수 | `012` | `int` |
| 정수형 | unsigned | `65536u`, `65536U` | `unsigned int` |
| 정수형 | long | `1234567L` | `long` |
| 실수형 | 부동소수점 표기 | `56.78`, `.5` | `double` |
| 실수형 | 지수 표기 | `5.678e1`, `.5e0` | `double` |
| 실수형 | float 접미사 | `0.25f`, `0.25F` | `float` |
| 문자열 | — | `"hello"`, `"x"` | `char[]` |

> **핵심:** `'x'`는 **문자** 리터럴 (타입 `int`, 값 120). `"x"`는 **문자열** 리터럴 (타입 `char[2]`, `'x'` + `'\0'` 저장). 둘은 다르다!

### 4.2 매크로 상수 (`#define`)

`#define`은 **기호 상수(symbolic constant)**를 만든다. 전처리기가 컴파일 전에 모든 해당 이름을 지정된 값으로 대치한다.

```c
#define HOURLY_WAGE 8350
#define PI 3.141592

wage = HOURLY_WAGE * hours;  // 전처리 후 → wage = 8350 * hours;
```

**규칙과 주의사항:**

```c
#define MAX_SIZE 100    // ✅ 올바름 — 끝에 세미콜론 없음
#define MAX_SIZE 100;   // ❌ 잘못됨 — 세미콜론까지 대치됨!

// 잘못된 정의로 전처리하면:
int arr[MAX_SIZE];  →  int arr[100;];  // 문법 오류!
```

```c
#define HOURLY_WAGE 8350
HOURLY_WAGE = 9000;           // ❌ 컴파일 에러 — 매크로는 변수가 아님
printf("HOURLY_WAGE\n");      // "HOURLY_WAGE" 출력 — 문자열 내부는 대치 안 됨
printf("%d\n", HOURLY_WAGE);  // 8350 출력 — 식별자는 대치됨
```

### 4.3 `const` 변수

`const`는 초기화 이후 값을 **변경할 수 없는** 변수를 선언한다.

```c
const int BUF_SIZE = 256;
const double PI = 3.141592;

BUF_SIZE = 128;  // ❌ 컴파일 에러
```

> **경고:** `const` 변수는 **선언 시 반드시 초기화**해야 한다. 초기화하지 않으면 쓰레기 값이 들어가고, 이후 값을 대입할 수 없어 무용지물이 된다.
> ```c
> const int size;      // ❌ 무의미 — 값을 저장할 수 없음
> const int size = 10; // ✅ 올바름
> ```

| 특성 | `#define` | `const` |
|:----|:---------:|:-------:|
| 타입 검사 | ❌ 없음 | ✅ 있음 |
| 유효 범위 | 전역 (정의 이후) | 블록 스코프 |
| 디버거 가시성 | ❌ 없음 (대치됨) | ✅ 있음 |
| 세미콜론 | ❌ 붙이지 않음 | ✅ 일반 문장 |

## 5. 형식 지정자

형식 지정자는 `printf`와 `scanf`에서 데이터를 해석하는 방식을 제어한다.

### 5.1 주요 지정자

| 지정자 | 타입 | 출력 예시 |
|:-----:|:----|:---------|
| `%d` | `int` (10진수) | `42` |
| `%u` | `unsigned int` | `100` |
| `%f` | `float`/`double` | `3.140000` |
| `%e` | `float`/`double` (지수 표기) | `3.140000e+00` |
| `%c` | `char` | `A` |
| `%s` | 문자열 | `hello` |
| `%x` | `int` (16진수, 소문자) | `2a` |
| `%X` | `int` (16진수, 대문자) | `2A` |
| `%#x` | `int` (16진수, `0x` 접두사) | `0x2a` |
| `%o` | `int` (8진수) | `52` |
| `%%` | 리터럴 `%` | `%` |

### 5.2 출력 폭과 정렬

| 형식 | 의미 | 예시 (`42`) |
|:----:|:----|:-----------|
| `%10d` | 우측 정렬, 폭 10 | `????????42` |
| `%-10d` | 좌측 정렬, 폭 10 | `42????????` |
| `%010d` | 0 채움, 폭 10 | `0000000042` |
| `%+d` | 부호 표시 | `+42` |
| `%.2f` | 소수점 이하 2자리 | `3.14` |
| `%.15f` | 소수점 이하 15자리 | `3.141592653589793` |
| `%08x` | 16진수, 0 채움, 폭 8 | `0000002a` |

(`?` = 공백)

```c
int no = 34, in = 124, out = 512;

printf("%5d", no);      //    34   (우측 정렬, 폭 5)
printf("%-6d", in);     // 124    (좌측 정렬, 폭 6)
printf("%06d", out);    // 000512 (0 채움, 폭 6)
```

> **팁:** `%Nd`는 N칸에 우측 정렬(기본). `%-Nd`는 좌측 정렬. `%0Nd`는 0으로 채움.

### 5.3 `scanf`

```c
int num;
scanf("%d", &num);  // & (주소 연산자)가 반드시 필요
```

> **경고:** `scanf`는 변수 자체가 아닌 **주소** (`&num`)를 요구한다. `&`를 빠뜨리면 정의되지 않은 동작이 발생한다.

## 6. 형 변환

### 6.1 묵시적 형 변환

피연산자의 타입이 다를 때, C는 작은 타입을 큰 타입으로 **승격(promotion)**한다.

```
char → int → long → float → double → long double
```

```c
int a = 5;
double b = 2.0;
double result = a + b;  // a가 double로 승격 → 7.0

double weight;
weight = 50;  // 50 (int) → 50.0 (double)

int length;
length = 12.5; // ⚠ 12.5 (double) → 12 (int), 데이터 손실!
```

### 6.2 명시적 형 변환 (캐스팅)

```c
int a = 7, b = 2;
double result;

result = a / b;          // int / int = int → 3 → 3.0
result = (double)a / b;  // double / int = double → 3.5
```

> **핵심:** `a / b`에서 둘 다 `int`이므로, 정수 나눗셈이 **먼저** 수행되어 결과가 3이 되고, 이것이 3.0으로 `double`에 대입된다. 실수 나눗셈을 하려면 **최소 하나의** 피연산자를 캐스팅해야 한다.

## 7. `sizeof` 연산자

`sizeof`는 데이터 타입, 변수, 리터럴의 **바이트 크기**를 컴파일 시점에 반환한다.

```c
printf("char:   %d\n", sizeof(char));    // 1
printf("short:  %d\n", sizeof(short));   // 2
printf("int:    %d\n", sizeof(int));     // 4
printf("long:   %d\n", sizeof(long));    // 4
printf("float:  %d\n", sizeof(float));   // 4
printf("double: %d\n", sizeof(double));  // 8
```

`sizeof`는 변수와 리터럴에도 사용할 수 있다:

```c
int a;
printf("%d\n", sizeof(a));      // 4
printf("%d\n", sizeof(3.14));   // 8  (double 리터럴)
printf("%d\n", sizeof(0.25f));  // 4  (float 리터럴)
printf("%d\n", sizeof('x'));    // 4  (C에서 문자 리터럴은 int!)
printf("%d\n", sizeof("x"));   // 2  ('x' + '\0')
```

> **참고:** C에서 문자 리터럴 `'x'`의 타입은 `int`(4바이트)이다 (`char`(1바이트)가 아님). 이것은 C만의 특성이며, C++에서는 `sizeof('x')`가 1이다.
