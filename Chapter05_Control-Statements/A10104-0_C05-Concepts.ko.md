## 1. 제어문 개요

기본적으로 C는 문장을 **순차적으로**(위에서 아래로) 실행한다. **제어문** 은 분기 또는 반복을 통해 이 흐름을 변경한다.

```
프로그램 흐름
├── 순차 구조   ← 기본 (위에서 아래로)
├── 조건 구조   ← 경로를 선택
├── 반복 구조   ← 블록을 반복
└── 분기 구조   ← 다른 위치로 이동
```

| 분류 | 목적 | C 키워드 |
|:----|:----|:--------|
| 조건문 | 조건에 따라 다른 블록 실행 | `if`, `else`, `else if`, `switch`, `case`, `default` |
| 반복문 | 조건이 참인 동안 블록 반복 | `for`, `while`, `do-while` |
| 분기문 | 블록 탈출 또는 건너뛰기 | `break`, `continue`, `goto`, `return` |

> **핵심:** **삼항 연산자**(`? :`)는 조건 *식(expression)*이지 문(statement)은 아니지만, 비슷한 역할을 한다: `max = (a > b) ? a : b;`

## 2. 조건문

### 2.1 `if`

`if`문은 조건이 참(0이 아닌 값)일 때**만** 블록을 실행한다.

**문법:**

```c
if (조건) {
    // 조건이 참(0이 아닌 값)일 때 실행
}
```

**실행 흐름:**

```
        ┌──────────┐
        │   조건    │
        └────┬─────┘
        참   │    거짓
             ▼         ──────► (건너뜀)
       ┌───────────┐
       │    문장    │
       └───────────┘
             │
             ▼
       (다음 코드)
```

```c
int score = 85;

if (score >= 60) {
    printf("합격!\n");
}
```

> **팁:** C에서 0이 아닌 모든 값은 **참(true)** 이고, 0은 **거짓(false)** 이다. 따라서 `if (1)`은 항상 실행되고, `if (0)`은 절대 실행되지 않는다.

### 2.2 복합문 / 블록

**복합문(compound statement, 블록)** 은 중괄호 `{}`로 여러 문장을 묶는 것이다.

```c
if (score >= 90) {
    printf("등급: A\n");
    printf("우수합니다!\n");
}
```

중괄호가 없으면 **첫 번째 문장만** `if`에 속한다:

```c
if (score >= 90)
    printf("등급: A\n");      // if에 속함
    printf("우수합니다!\n");    // 항상 실행됨! (if와 무관)
```

> **경고:** 단일 문장이라도 항상 중괄호 `{}`를 사용하라. 나중에 코드를 추가할 때 버그를 방지하고, 의도를 명확히 한다.

### 2.3 `if-else`

참일 때와 거짓일 때, 두 가지 경로를 제공한다.

**문법:**

```c
if (조건) {
    // 참일 때 실행
} else {
    // 거짓일 때 실행
}
```

**실행 흐름:**

```
        ┌──────────┐
        │   조건    │
        └────┬─────┘
   참        │      거짓
     ▼       │        ▼
 ┌───────┐   │   ┌────────┐
 │ if    │   │   │  else  │
 │ 블록   │   │   │  블록   │
 └───────┘   │   └────────┘
     │        │        │
     └────────┴────────┘
              ▼
        (다음 코드)
```

```c
int num = 7;

if (num % 2 == 0) {
    printf("%d은(는) 짝수\n", num);
} else {
    printf("%d은(는) 홀수\n", num);
}
```

**`if-else` vs 별개의 `if` 두 개:**

```c
// if-else: 정확히 하나의 분기만 실행 (상호 배타적)
if (x > 0) {
    printf("양수\n");
} else {
    printf("양수 아님\n");
}

// 별개의 if 두 개: 각 조건이 독립적으로 검사됨
if (x > 0) {
    printf("양수\n");
}
if (x % 2 == 0) {
    printf("짝수\n");
}
```

### 2.4 `else if`

`else if` 사다리는 조건을 **순차적으로** 검사한다. 참인 조건을 찾으면 해당 블록을 실행하고, 나머지 조건은 모두 **건너뛴다**.

**문법:**

```c
if (조건1) {
    // 블록 1
} else if (조건2) {
    // 블록 2
} else if (조건3) {
    // 블록 3
} else {
    // 기본 블록 (선택 사항)
}
```

마지막 `else`는 **선택 사항** 이지만, 예외 상황을 처리하기 위해 권장된다.

```c
int score;
printf("점수 입력: ");
scanf("%d", &score);

if (score >= 90) {
    printf("등급: A\n");
} else if (score >= 80) {
    printf("등급: B\n");
} else if (score >= 70) {
    printf("등급: C\n");
} else if (score >= 60) {
    printf("등급: D\n");
} else {
    printf("등급: F\n");
}
```

> **핵심:** 조건이 순서대로 검사되므로, `score >= 80`은 암묵적으로 `80 <= score < 90`을 의미한다 (이미 `score >= 90`이 거짓으로 판정되었으므로). `score >= 80 && score < 90`처럼 쓸 **필요가 없다**.

### 2.5 중첩 `if`

`if` 또는 `else` 블록 안에 또 다른 `if`문을 넣는 것이다.

```c
int x, y;
scanf("%d %d", &x, &y);

if (x == 0 && y == 0) {
    printf("원점\n");
} else if (x == 0) {
    printf("y축 위\n");
} else if (y == 0) {
    printf("x축 위\n");
} else {
    if (x > 0 && y > 0) {
        printf("제1사분면\n");
    } else if (x < 0 && y > 0) {
        printf("제2사분면\n");
    } else if (x < 0 && y < 0) {
        printf("제3사분면\n");
    } else {
        printf("제4사분면\n");
    }
}
```

> **경고:** 깊게 중첩된 `if`문은 가독성이 떨어진다. `else if` 사다리나 조기 반환(early return)을 사용하여 구조를 평탄하게 만드는 것이 좋다.

### 2.6 독립된 여러 `if`

조건이 **상호 배타적이지 않을 때** 는 별개의 `if`문을 사용한다. 각 조건이 독립적으로 평가된다.

```c
int temp = 35;

if (temp > 30) {
    printf("더움\n");       // 출력됨
}
if (temp > 20) {
    printf("따뜻함\n");     // 역시 출력됨
}
if (temp > 10) {
    printf("온화함\n");     // 역시 출력됨
}
```

세 조건 모두 참이므로 세 메시지가 모두 출력된다. 이를 `else if`와 비교하면, `else if`에서는 **첫 번째** 일치 블록만 실행된다.

### 2.7 `switch`

`switch`문은 **정수 또는 문자 식** 의 값에 따라 블록을 선택한다. 여러 이산적(discrete) 값을 비교할 때 긴 `else if` 사다리보다 깔끔하다.

**문법:**

```c
switch (식) {
case 값1:
    // 문장
    break;
case 값2:
    // 문장
    break;
// ...
default:
    // 일치하는 case가 없을 때 실행
}
```

```c
char unit;
printf("온도 단위 (C/F): ");
scanf(" %c", &unit);

switch (unit) {
case 'C':
case 'c':
    printf("섭씨\n");
    break;
case 'F':
case 'f':
    printf("화씨\n");
    break;
default:
    printf("잘못된 단위!\n");
}
```

**핵심 규칙:**

| 규칙 | 설명 |
|:----|:----|
| 식의 타입 | `int` 또는 `char` (정수형)이어야 한다. `float`, `double`, 문자열은 **불가**. |
| case 레이블 | **상수** 값이어야 한다 (변수 불가). |
| `break` | **폴스루(fall-through)** 를 방지하려면 반드시 필요. 없으면 다음 case로 계속 실행. |
| `default` | 선택 사항. 어떤 case에도 일치하지 않는 값을 처리. |
| 중복 case | 불가. 각 case 레이블은 고유해야 한다. |

**폴스루 동작 (`break` 누락):**

```c
int n = 1;
switch (n) {
case 1:
    printf("one\n");      // 출력됨
    // break 없음!
case 2:
    printf("two\n");      // 역시 출력됨! (폴스루)
    break;
case 3:
    printf("three\n");
    break;
}
// 출력: one
//       two
```

> **경고:** `break`를 빠뜨리는 것은 C에서 가장 흔한 버그 중 하나이다. 의도적인 폴스루(`'C'`와 `'c'`를 묶는 경우 등)는 명확히 표시해야 한다.

**`switch`로 정수 범위 처리 (성적 등급 매기기):**

```c
int score = 85;
switch (score / 10) {
case 10:
case 9:
    printf("A\n");
    break;
case 8:
    printf("B\n");
    break;
case 7:
    printf("C\n");
    break;
case 6:
    printf("D\n");
    break;
default:
    printf("F\n");
}
```

### 2.8 `if-else`와 `switch` 비교

| 특성 | `if-else` | `switch` |
|:----|:---------|:---------|
| 조건 타입 | 임의의 논리 식 | 정수/문자 동등 비교만 가능 |
| 범위 검사 | `score >= 90` | 직접 불가 (`score / 10` 사용) |
| 다중 값 | `if (x == 1 \|\| x == 2)` | `case 1: case 2:` (폴스루) |
| 실수/문자열 | 지원 | 미지원 |
| 가독성 | 범위/복합 조건에 적합 | 많은 이산 값에 적합 |
| 성능 | 순차 검사 | 점프 테이블 사용 가능 (case가 많을 때 빠름) |

> **팁:** 하나의 변수를 여러 **상수 정수/문자 값** 과 비교할 때는 `switch`를 사용하라. 범위, 실수 비교, 복합 조건에는 `if-else`를 사용하라.

## 3. 반복문

### 3.1 `for`

`for` 반복문은 **반복 횟수가 미리 정해져 있을 때** 가장 적합하다.

**문법:**

```c
for (초기화; 조건; 증감) {
    // 반복 본체
}
```

**실행 흐름:**

```
   초기화
    │
    ▼
┌─► 조건 ──거짓──► 종료
│    │
│    참
│    ▼
│  반복 본체
│    │
│    ▼
└── 증감
```

1. **초기화** — 반복문 시작 전 한 번만 실행.
2. **조건** — 매 반복 전에 검사. 거짓이면 반복문 종료.
3. **반복 본체** — 조건이 참이면 실행.
4. **증감** — 매 반복 후 실행되고, 다시 2단계로 돌아감.

```c
// 1부터 5까지 출력
for (int i = 1; i <= 5; i++) {
    printf("%d ", i);
}
// 출력: 1 2 3 4 5
```

**자주 쓰는 패턴:**

```c
// 역순 카운트
for (int i = 10; i >= 1; i--) {
    printf("%d ", i);
}

// 2씩 증가
for (int i = 0; i <= 10; i += 2) {
    printf("%d ", i);
}

// 1~100 합계
int sum = 0;
for (int i = 1; i <= 100; i++) {
    sum += i;
}
printf("합계: %d\n", sum);  // 5050
```

**`for`의 구성 요소 생략:**

```c
int i = 0;
for ( ; i < 5; ) {      // 초기화와 증감 생략
    printf("%d ", i);
    i++;
}

for (;;) {               // 무한 반복 (모든 요소 생략)
    // break로 탈출
}
```

> **경고:** `break`나 `return` 없는 무한 반복문은 영원히 실행되어 프로그램이 멈춘다.

### 3.2 `while`

`while` 반복문은 **종료 조건이 실행 중 입력이나 이벤트에 따라 결정될 때** 가장 적합하다.

**문법:**

```c
while (조건) {
    // 반복 본체
}
```

**실행 흐름:**

```
    ┌─► 조건 ──거짓──► 종료
    │    │
    │    참
    │    ▼
    │ 반복 본체
    │    │
    └────┘
```

조건은 매 반복 **전에** 검사된다. 처음부터 거짓이면 본체가 한 번도 실행되지 않는다.

```c
// 감시값(sentinel) 제어 반복: -1이면 종료
int num, sum = 0, count = 0;

printf("정수 입력 (종료: -1): ");
scanf("%d", &num);

while (num != -1) {
    sum += num;
    count++;
    printf("정수 입력 (종료: -1): ");
    scanf("%d", &num);
}

if (count > 0) {
    printf("합계: %d, 평균: %.2f\n", sum, (double)sum / count);
}
```

**`break`를 이용한 무한 반복:**

```c
while (1) {
    int n;
    printf("양수를 입력하세요: ");
    scanf("%d", &n);
    if (n > 0) {
        printf("입력한 값: %d\n", n);
        break;
    }
    printf("잘못된 입력! 다시 시도하세요.\n");
}
```

### 3.3 `do-while`

`do-while` 반복문은 본체를 **최소 한 번 반드시 실행** 한 후 조건을 검사한다.

**문법:**

```c
do {
    // 반복 본체
} while (조건);  // <-- 세미콜론 필수!
```

**실행 흐름:**

```
    ┌───────┐
    │       ▼
    │    반복 본체(일단 실행)
    │       │
    │       ▼
    │      조건 ──거짓──► 종료
    │       │
    └─────  참
```

```c
int choice;
do {
    printf("1. 실행  2. 종료\n");
    printf("선택: ");
    scanf("%d", &choice);
    if (choice < 1 || choice > 2) {
        printf("잘못된 선택!\n");
    }
} while (choice != 2);
printf("종료합니다!\n");
```

> **경고:** `do-while`문에서 `while(조건);` 뒤의 **세미콜론** 을 빠뜨리지 말 것. 흔한 문법 오류이다.

**최소 한 번 실행 보장:**

```c
// do-while: x > 10이어도 본체가 한 번은 실행됨
int x = 100;
do {
    printf("x = %d\n", x);  // 한 번 출력됨
} while (x < 10);

// while: x > 10이면 본체가 실행되지 않음
while (x < 10) {
    printf("x = %d\n", x);  // 출력되지 않음
}
```

### 3.4 중첩 반복문

반복문 안에 또 다른 반복문을 넣는 것이다. **내부 반복문은** 외부 반복문의 한 번의 반복마다 **모든 반복을 완료** 한다.

**좌측 정렬 삼각형 패턴:**

```c
int n = 4;
for (int i = 1; i <= n; i++) {        // 행
    for (int j = 1; j <= i; j++) {    // 열
        printf("*");
    }
    printf("\n");
}
```

출력:
```
*
**
***
****
```

**우측 정렬 삼각형 패턴:**

```c
int n = 4;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= n - i; j++) {   // 앞쪽 공백
        printf(" ");
    }
    for (int j = 1; j <= i; j++) {       // 별
        printf("*");
    }
    printf("\n");
}
```

출력:
```
   *
  **
 ***
****
```

**구구단 (2차원 출력):**

```c
for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= 9; j++) {
        printf("%3d", i * j);
    }
    printf("\n");
}
```

**부호 교대 합계 (`1-2+3-4+...+9-10`):**

```c
int sum = 0;
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 1) {
        sum += i;
    } else {
        sum -= i;
    }
}
printf("결과: %d\n", sum);  // -5
```

### 3.5 `for` vs `while` vs `do-while` 비교

| 특성 | `for` | `while` | `do-while` |
|:----|:------|:--------|:-----------|
| 적합한 경우 | 반복 횟수가 정해진 경우 | 횟수 미정, 이벤트 기반 | 최소 한 번 실행 필요 |
| 조건 검사 시점 | 매 반복 전 | 매 반복 전 | 매 반복 후 |
| 최소 실행 횟수 | 0 | 0 | 1 |
| 대표 용도 | 카운팅, 배열 순회 | 감시값 반복, 입력 읽기 | 입력 검증, 메뉴 |
| 문법 | `for (초기화; 조건; 증감)` | `while (조건)` | `do { } while (조건);` |

> **핵심:** `for` 반복문은 언제나 `while` 반복문으로, 그 반대도 변환할 수 있다. 의도를 가장 명확히 전달하는 것을 선택하라.

```c
// 동치인 반복문:

// for 버전
for (int i = 0; i < 5; i++) {
    printf("%d ", i);
}

// while 버전
int i = 0;
while (i < 5) {
    printf("%d ", i);
    i++;
}
```

## 4. 분기문

### 4.1 `break`

`break`는 **가장 안쪽의 반복문** 또는 `switch`문을 즉시 탈출한다.

```c
for (int i = 1; i <= 100; i++) {
    if (i * i > 50) {
        printf("i^2 > 50인 첫 번째 i: %d\n", i);
        break;  // for 반복문 탈출
    }
}
```

중첩 반복문에서 `break`는 **내부** 반복문만 탈출한다:

```c
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) {
            break;  // 내부 반복문만 탈출
        }
        printf("(%d,%d) ", i, j);
    }
}
// 출력: (0,0) (1,0) (2,0)
```

### 4.2 `continue`

`continue`는 **현재 반복의 나머지 부분을 건너뛰고** 다음 반복으로 넘어간다 (`while`/`do-while`에서는 조건 검사로, `for`에서는 증감 단계로 이동).

```c
// 홀수만 출력
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;  // 짝수 건너뜀
    }
    printf("%d ", i);
}
// 출력: 1 3 5 7 9
```

> **팁:** `for` 반복문에서 `continue`는 **증감식**(`i++`)으로 이동한다 (조건이 아님). `while` 반복문에서는 **조건 검사** 로 바로 이동한다. 따라서 `continue` 전에 반복 변수가 갱신되었는지 확인하지 않으면 무한 반복에 빠질 수 있다.

### 4.3 `goto`

`goto`는 레이블이 붙은 문장으로 무조건 이동한다. 코드를 추적하기 어렵게 만들므로(스파게티 코드) **일반적으로 사용을 피한다**.

```c
for (int i = 0; i < 10; i++) {
    for (int j = 0; j < 10; j++) {
        if (i + j == 15) {
            goto found;  // 두 반복문 모두 탈출
        }
    }
}
found:
printf("완료\n");
```

> **경고:** 거의 모든 경우에 `goto`를 피하라. 유일하게 허용되는 사용 사례는 깊게 중첩된 반복문에서 한 번에 탈출하는 것이다. `break`, `continue`, 또는 함수를 대신 사용하라.

### 4.4 `return`

`return`은 **현재 함수를 종료** 하고 선택적으로 값을 반환한다. `main`에서 사용하면 프로그램이 종료된다.

```c
int find_first_negative(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        if (arr[i] < 0) {
            return arr[i];  // 함수 즉시 종료
        }
    }
    return 0;  // 음수 없음
}
```

## 5. 자주 사용되는 패턴

### 5.1 카운터와 누적기

**카운터** 는 무언가가 몇 번 발생했는지 추적한다. **누적기** 는 누적 합계를 추적한다.

```c
int sum = 0;    // 누적기
int count = 0;  // 카운터
int num;

printf("정수를 입력하세요 (0이면 종료):\n");
while (1) {
    scanf("%d", &num);
    if (num == 0) {
        break;
    }
    sum += num;
    count++;
}

if (count > 0) {
    printf("합계: %d\n", sum);
    printf("개수: %d\n", count);
    printf("평균: %.2f\n", (double)sum / count);
}
```

> **핵심:** 실수 평균이 필요하면 나누기 전에 `double`로 캐스팅하라: `(double)sum / count`. 정수 나눗셈 `sum / count`는 소수 부분을 버린다.

### 5.2 최댓값/최솟값 추적

최댓값/최솟값을 **첫 번째 입력값** 으로 초기화한 후, 새 값이 들어올 때마다 갱신한다.

```c
int num, max, min, count = 0;

while (1) {
    printf("값 입력 (0이면 종료): ");
    scanf("%d", &num);
    if (num == 0) {
        break;
    }

    if (count == 0) {
        max = min = num;  // 첫 번째 값으로 둘 다 초기화
    } else {
        if (num > max) max = num;
        if (num < min) min = num;
    }
    count++;
}

if (count > 0) {
    printf("MAX: %d, MIN: %d\n", max, min);
}
```

**삼항 연산자** 로 간결하게 최댓값/최솟값을 구할 수도 있다:

```c
int a, b, c;
scanf("%d %d %d", &a, &b, &c);

int max = (a > b) ? a : b;
max = (max > c) ? max : c;
printf("Max: %d\n", max);
```

### 5.3 입력 검증 반복

`do-while`을 사용하여 유효한 입력을 받을 때까지 반복한다.

```c
int score;
do {
    printf("점수 입력 (0-100): ");
    scanf("%d", &score);
    if (score < 0 || score > 100) {
        printf("오류! 점수는 0-100이어야 합니다.\n");
    }
} while (score < 0 || score > 100);

printf("유효한 점수: %d\n", score);
```

### 5.4 메뉴 기반 프로그램

`while(1)` + `switch` + `do-while`을 조합하여 대화형 메뉴를 구현한다.

```c
#include <stdio.h>

int main() {
    char menu;

    while (1) {
        printf("========================\n");
        printf("1. 옵션 A\n");
        printf("2. 옵션 B\n");
        printf("3. 종료\n");
        printf("========================\n");

        // do-while로 입력 검증
        do {
            printf("선택: ");
            scanf(" %c", &menu);
            if (menu < '1' || menu > '3') {
                printf("잘못된 입력! 다시 선택하세요.\n");
            }
        } while (menu < '1' || menu > '3');

        if (menu == '3') {
            printf("종료합니다!\n");
            break;
        }

        switch (menu) {
        case '1':
            printf("옵션 A를 선택했습니다\n");
            break;
        case '2':
            printf("옵션 B를 선택했습니다\n");
            break;
        }
    }

    return 0;
}
```

> **팁:** `scanf`로 `char`를 읽을 때 `" %c"` (앞에 공백)를 사용하면 입력 버퍼에 남아 있는 공백/개행 문자를 건너뛸 수 있다.
